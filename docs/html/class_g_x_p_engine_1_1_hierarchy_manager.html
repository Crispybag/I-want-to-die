Detailed Description

If you are getting strange bugs because you are calling Destroy during the Update loop, you can use this class to do this more cleanly: when using HierarchyManager.Instance.LateDestroy, all these hierarchy changes will be made after the update loop is finished. Similarly, you can use HierarchyManager.Instance.LateCall to postpone a certain method call until after the update loop.
